# User Stories

| Story | Project | As a | I want | So that | Risk |
| --- | --- | --- | --- | --- | --- |
| User login | Debug Diner | User/ Employee/ Admin | Log in with my email and password | i can access my information | High |
| User make reservation | Debug Diner | User | To make a reservation for myself | I am assured of a table in the restaurant | Medium |
| User view reservations | Debug Diner | User | To view all my current reservations | I have an overview of my own reservations | Low |
| User edit reservations | Debug Diner | User | To edit a reservation i made | I can adjust information about the reservation | Low |
| User see user info | Debug Diner | User/ Employee/ Admin | To view my own information | I know what is stored by the application | Low |
| User edit user info | Debug Diner | User/ Employee/ Admin | To edit my own information | I can change my allergen information, address, password, and the like | Low |
| Employee see all reservations | Debug Diner | Employee/ Admin | To see all reservations | I can manage, edit, or cancel existing reservations | Medium |
| Employee add new reservation | Debug Diner | Employee/ Admin | To make a reservation for a customer | They don’t have to do it themselves | High |
| Admin add new employee | Debug Diner | Admin | To make or delete an employee account | when I hire or fire an employee they can immediately edit and manage reservations | High |
| Admin view users | Debug Diner | Admin | To see all registered users | I have an idea who and what is coming over | High |
