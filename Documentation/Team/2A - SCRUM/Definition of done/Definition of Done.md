# Definition of Done - Debug Diner

Een user story wordt als **Done** beschouwd wanneer aan ALLE onderstaande criteria is voldaan:

| # | Criterium | Controle |
|---|---|---|
| 1 | Code werkt correct in C# (.NET 10.0) | ✅ |
| 2 | Unit tests geschreven én geslaagd (xUnit + FluentAssertions) | ✅ |
| 3 | Code gereviewed via Pull Request op GitHub | ✅ |
| 4 | Alle acceptatiecriteria van de story zijn gehaald | ✅ |
| 5 | Geen bekende bugs | ✅ |
| 6 | Urenregistratie bijgewerkt in Notion | ✅ |
| 7 | Branch gemerged naar `main` na goedkeuring PR | ✅ |

## Branch & PR werkwijze

- Elke taak krijgt een eigen branch, vernoemd naar de taaknaam
- Code moet lokaal getest zijn vóór een Pull Request
- Pull Requests worden uiterlijk vrijdagavond aangemaakt
- Minimaal één teamlid reviewt de PR vóór merge
- Review-comments zijn constructief en oplossingsgericht
